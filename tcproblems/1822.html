Problem Statement for LetterFreq

Problem Statement

    It turns out that, given a large enough document, it is usually possible to determine the language in which the document is written solely by looking at the letter frequencies in the document. Your task is to write a helper method for a program that determines the language in which a document is written. You will be given a String[], doc, which represents the document. You should return a int[] with 26 elements, each of which represents the number of occurrences of a given letter found in the document. The first element of the returned int[] is the number of A's, the second is the number of B's, and so on. Uppercase and lowercase letters should be treated the same. Characters other than letters ('A'-'Z' and 'a'-'z') should be ignored.
 

Definition

    
Class:LetterFreq
Method:getFreqs
Parameters:String[]
Returns:int[]
Method signature:int[] getFreqs(String[] doc)
(be sure your method is public)
    
 

Constraints

-doc will contain between 1 and 50 elements, inclusive.
-Each element of doc will contain between 1 and 50 characters, inclusive.
-Each character in each element of doc will have ASCII value between 32 and 126, inclusive.
